Create navigational links within a web site

There are several techniques to nivigate within a web site. The semantic tag for navigation is the <nav>, tag. You can navigate through your web site by using relative addressing. For our examples we will use the unordered list tag (<ul>).

  • page to page within the same folder
  • page to page between folders
  • staying on the current page
  • going to a specific place on the current page (on page navigation)

navigational menus are placed in your <header> semantic area. They may also be places in your <footer> area as a sitemap. The nav tag like other semantic tags should have a <hx> tag. HOWEVER this header is optional. If you omit the header tag you will recieve a document outliner error: Untitled section. Until you are shown how to avoiod this error message, it will be acceptable on your document outline.

CSS Coding styles Summary

Browser Defualts (no css) you have no control over the styling
Inline this is for a single specific element (tag) instance. your css will be specified in a parameter within the element. This css affects ONLY the element tag that contains the specification. style="rule; [rule; ...]" style="color:red; width:50px;"
Embedded the specificaton of css as embedded happens in your <head> area. this css affects all similar elements on the web page where the css is coded. <stylel> selector(rule; [rule; ...])
[selector(rule; [rule; ...]) ...] </style>		 <stylel> p (color:red; width:50px;)
h1 (font-style:italics) </style>
Link (stylesheet) this is the preferred method of doing css. You place your css in a seperate physical file. This file can then be used on ANY web page in your web site. The link to this file will be done in your <head> area. The css in this file is similar to embedded EXEPT there is no style tag selector(rule; [rule; ...])
[selector(rule; [rule; ...]) ...]		 p (color:red; width:50px;)
h1 (font-style:italics)
